Schema(스키마)의 개념
• MongoDB에서의 Schema(스키마)란?
§ 스키마(Schema) 는 데이터베이스에서 데이터의 구조(structure), 속성(attributes),
제약조건(constraints) 등을 정의하는 설계도와 같은 개념
§ 전통적인 관계형 데이터베이스(RDBMS)에서는 고정된 스키마를 따르지만, MongoDB는 
스키마리스(schema-less) 구조를 지원
• MongoDB에서의 스키마의 특징
§ MongoDB는 유연한 문서 기반(Document-oriented) 구조이기 때문에 같은 컬렉션 안의 
문서들이 서로 다른 필드를 가질 수 있음.
§ 하지만, Mongoose 같은 ODM(Object Data Modeling) 라이브러리를 사용하면, Node.js와 같은 
환경에서도 명시적인 스키마를 정의할 수 있음.

데이터 모델링 개념
• 데이터 모델링(Data Modeling)은 데이터를 효율적이고 일관되게 저장하고 접근할 수 있도록 구조를 
설계하는 과정
• MongoDB에서는 문서(Document) 단위로 데이터를 모델링하며, 관계형 데이터베이스의 정규화 
접근과는 다른 방식이 필요함.
• 스키마 유연성: 필드와 데이터 타입이 고정되지 않음.
• 중첩 문서(Embedded Document)와 배열을 자유롭게 사용 가능.
• 관계 표현 방식이 전통적인 DBMS와 다름:
§ RDBMS: 테이블 + Foreign Key
§ MongoDB: 임베딩 vs 레퍼런싱

Embedding Pattern
•
관련된 데이터를 하나의 문서에 중첩하여 
저장하는 방식
•
조회 시 조인 없이 한 번의 read로 모든 관련 
정보 접근 가능
•
읽기(Read)가 많고, 데이터 크기가 작거나 
변경이 적을 때 유리
•
장점:
⇒빠른 조회 (Join 불필요)
⇒데이터 일관성 보장
•
단점:
⇒중복 가능성 있음 (다른 문서에도 같은 구조가 있을 수 
있음)
⇒문서 크기 제한(16MB)을 초과할 수 있음

Reference Pattern
•
관련 데이터를 다른 컬렉션에 분리하고,
ObjectId를 참조(Reference)하여 연결하는 방식
•
관계형 DB의 Foreign Key 개념과 유사
•
쓰기/수정이 잦거나, 데이터가 커질 가능성이 
있을 때 적합
•
장점:
⇒데이터 중복 최소화
⇒각 문서를 독립적으로 관리 및 수정 가능
•
단점:
⇒조인 기능이 제한적이므로, 여러 번의 쿼리(fetch) 필요
⇒복잡한 조회 시 성능 저하 가능

Bucket Pattern
MongoDB Schema
• 고빈도 데이터(이벤트/로그 등)를 일정 기준으로 묶어 하나의 
문서에 저장하는 방식

• MongoDB의 문서 수를 줄이고, 인덱스 부담을 완화할 수 
있음
• 주요 특징
§ "버킷(bucket)"은 시간, 사용자, 카테고리 등의 기준으로 그룹화
§ 반복되는 데이터 구조를 하나의 문서 배열로 묶어 저장
§ 데이터 조회 시 일부만 가져오거나 필터링할 수 있음
